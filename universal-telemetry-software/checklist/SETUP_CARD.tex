\section*{4.5 -- CAR RPi: NATIVE SYSTEMD (RECOMMENDED)}
% ─────────────────────────────────────────────────────────────────────────────

\noindent
\textit{Running telemetry as a native systemd service on the car is more reliable than Docker:
faster cold boot ($<$3 s), no overlay2 corruption on power loss, automatic process restart.
No Redis needed -- the core CAN $\to$ UDP path runs entirely in-process.}

\gobox{One-time setup (internet required)}

\noindent
\begin{tabular}{p{126mm}}
  Install \cmd{uv}: \cmd{curl -LsSf https://astral.sh/uv/install.sh | sh} \\[1mm]
  Pre-build the Python venv (needs internet -- do this in the pits, not on the grid): \\
  \cmd{cd \textasciitilde/daq-radio/universal-telemetry-software \&\& uv sync} \\[1mm]
  Install the service (stamps current git hash into unit file): \\
  \cmd{sed -i "s/GIT\_HASH=unknown/GIT\_HASH=\$(git rev-parse --short HEAD)/" deploy/car-telemetry.service} \\
  \cmd{sudo cp deploy/car-telemetry.service /etc/systemd/system/} \\
  \cmd{sudo systemctl daemon-reload} \\
  \cmd{sudo systemctl enable car-telemetry} \\
  \cmd{sudo systemctl start car-telemetry} \\[1mm]
  \textbf{PASS:} \cmd{systemctl status car-telemetry} shows \cmd{active (running)} \\
\end{tabular}

\vspace{1mm}
\gobox{Daily ops}

\noindent
\begin{tabular}{p{126mm}}
  View logs: \cmd{journalctl -u car-telemetry -f} \\
  Restart: \cmd{sudo systemctl restart car-telemetry} \\
  Stop: \cmd{sudo systemctl stop car-telemetry} \\[1mm]
  \textit{Note: car does not use Docker -- it runs via systemd instead.} \\
\end{tabular}

\section*{9.5 -- ECU TIME SYNC (Car RPi only)}
% ─────────────────────────────────────────────────────────────────────────────

\noindent
\begin{tabular}{p{126mm}}
  \textbf{The car RPi has no NTP -- system clock resets to 1970 on boot without a time source.} \\
  CAN frames are timestamped on the car RPi before UDP transmission; the base/MacBook just \\
  writes those timestamps into TimescaleDB unchanged. Getting the car RPi's clock right \\
  is therefore the only thing that matters for correct TimescaleDB timestamps. \\[1mm]
  The ECU has a battery-backed DS3231 RTC and broadcasts the real epoch time on \textbf{CAN ID 1999} \\
  (\cmd{VCU\_Timestamp}) once per second. The car RPi intercepts this and corrects its clock. \\[1mm]
  \textbf{Car data is held (not dropped) until one of three conditions is met:} \\
\end{tabular}

\vspace{1mm}
\noindent
\begin{tabularx}{\linewidth}{p{4mm}p{30mm}X}
  \rowcolor{tblhd}
  & \textbf{Source} & \textbf{How} \\
  1 & ECU RTC & ECU broadcasts CAN ID 1999 at 1 Hz -- RPi uses it automatically within $\sim$1s of CAN coming up. \textit{Primary -- most reliable.} \\
  2 & RPi system clock & If RPi clock is already past 2026-03-22 (NTP synced, or manually set previously), data flows immediately. \\
  3 & Manual set & User sets RPi clock via status page -- see below. \\
\end{tabularx}

\vspace{1mm}
\gobox{Verify Sync}

\noindent
\begin{tabular}{p{126mm}}
  \cmd{docker logs daq-telemetry $\vert$ grep -E "time sync$\,$\textbar{}trusting RPi"} \\[1mm]
  \textbf{PASS (ECU RTC):} \cmd{ECU time sync: epoch=1742... offset=+Xs} \\
  \textbf{PASS (system clock):} \cmd{System clock past 2026-03-22, trusting RPi clock} \\
  \textbf{FAIL:} no match -- data is being held. If ECU is on the bus, wait up to 2s for first broadcast. \\
\end{tabular}

\vspace{1mm}
\warnbox{No ECU + no NTP = data held indefinitely. Use manual set below.}

\vspace{1mm}
\checkbox{Manual Time Set (bench test / no ECU)}

\noindent
\begin{tabular}{p{126mm}}
  Open the status page on the \textbf{car RPi}: \\[1mm]
  \textbf{On radio subnet:} \cmd{http://10.71.1.10:8080} \\
  \textbf{Via hotspot (phone/laptop):} \cmd{http://192.168.10.1:8080} \\[1mm]
  If clock is not synced, an amber banner is shown at the top with a datetime input. \\
  \quad 1.\ Click \textbf{Now\,$\circlearrowleft$} -- pre-fills with your laptop's current time (assumed correct). \\
  \quad 2.\ Click \textbf{Set RPi Clock} -- sends \cmd{POST /set-time} to the status server. \\
  \quad 3.\ The container runs privileged, so \cmd{date -s} sets the host kernel clock directly. \\
  \quad 4.\ Within 1s, \cmd{\_try\_auto\_sync()} detects clock $>$ 2026-03-22 and unblocks data. \\[1mm]
  \textbf{PASS:} banner turns green. Check logs: \cmd{grep "trusting RPi clock"}. \\[1mm]
  \textit{Note:} if you only have SSH and no browser, run on the RPi directly: \\
  \cmd{sudo date -s "2026-03-22 14:35:00"} \\[2mm]
  \textit{Base RPi clock:} the base RPi's system clock does \textbf{not} affect data timestamps -- \\
  the car already stamped every frame before UDP transmission, so Grafana and TimescaleDB \\
  will show correct times even if the base is in 1970. \\
  Nice-to-have: set base clock for clean container logs. \\
  Same method -- \cmd{http://10.71.1.20:8080} or \cmd{http://192.168.20.1:8080} via hotspot. \\
\end{tabular}
